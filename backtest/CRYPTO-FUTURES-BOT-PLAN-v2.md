# Crypto Futures Bot — Переосмысление через Skills
## Product Brainstorming × Risk Assessment

> Применены фреймворки: JTBD, Opportunity Solution Tree, Assumption Testing,  
> OODA Loop, Reverse Brainstorming, Risk Register Matrix

---

## ШАГ 1 — FRAME (Зачем это вообще?)

Прежде чем строить ботов, ответим на неудобный вопрос:

> **"Кто имеет эту задачу, и что они делают сейчас без бота?"**

**Ответ честно:** Соло-трейдер с капиталом, который хочет пассивный рост,  
но сейчас либо торгует руками (стресс + пропущенные сделки) либо держит в стейкинге (5-8% APR, не хватает).

---

## ШАГ 2 — JOBS-TO-BE-DONE

Не "что строим", а "какую работу нанимают эти боты выполнять":

```
JTBD #1 — Финансовый
"Когда у меня есть свободный капитал,
 я хочу, чтобы он рос 24/7 без моего участия,
 чтобы я мог заниматься другими вещами, не упуская рыночные возможности."

JTBD #2 — Психологический
"Когда рынок падает или растёт,
 я хочу убрать эмоции из торговых решений,
 чтобы не делать импульсивных сделок, о которых потом жалею."

JTBD #3 — Операционный
"Когда открыта позиция,
 я хочу автоматически управлять стопами и тейками,
 чтобы не просиживать у экрана и не терять сон."
```

**Неудобный вопрос (Assumption Testing):**  
Если JTBD #1 — это "рост без участия", то почему не купить ETF или индекс?  
→ **Ответ:** Крипто-рынок неэффективен. Алго-стратегии там дают 3-10x больше, чем buy-and-hold. Это доказуемо на данных 2020-2024.

## ШАГ 3 — OPPORTUNITY SOLUTION TREE

```
ЦЕЛЕВОЙ РЕЗУЛЬТАТ: Стабильный рост капитала >30% годовых с DD < 20%
│
├── ВОЗМОЖНОСТЬ A: Захватить направленные движения рынка
│   ├── Решение A1: EMA-кроссовер + ADX-фильтр (классика)
│   │   └── Эксперимент: Backtest 2020-2024, проверить Sharpe > 1.2
│   ├── Решение A2: Breakout по ATR с объёмным подтверждением
│   │   └── Эксперимент: Сравнить win rate vs A1 на одних данных
│   └── Решение A3: ML-предсказание через neural-trader (LSTM)
│       └── Эксперимент: Out-of-sample на 2024 Q1-Q2 данных
│
├── ВОЗМОЖНОСТЬ B: Зарабатывать независимо от направления
│   ├── Решение B1: Funding Rate Arbitrage (спот + шорт перп)
│   │   └── Эксперимент: Посчитать средний APR за 2022-2024 (bull + bear)
│   └── Решение B2: Stat Arb BTC/ETH spread (Z-score)
│       └── Эксперимент: Cointegration test, half-life спреда
│
└── ВОЗМОЖНОСТЬ C: Автоматизировать боковик
    └── Решение C1: Grid Bot с автодетекцией диапазона через ADX < 20
        └── Эксперимент: % времени рынок в боковике (исторически ~55%)
```

**Вывод OST:** Каждая Возможность — отдельная гипотеза с конкретным экспериментом.  
Не строить всё сразу. Валидировать по одному.

## ШАГ 4 — ASSUMPTION TESTING (Вскрываем ложные убеждения)

Это самое важное. Большинство людей теряют деньги именно здесь.

### Допущение #1 (КРИТИЧЕСКОЕ)
> "Мой бэктест предсказывает реальную доходность"

**Риск:** ЛОЖНОЕ. Это убивает большинство ботов.  
**Почему ложное:**
- Транзакционные издержки занижены (slippage, funding, комиссии = 0.04-0.1% за сделку)
- Look-ahead bias — случайно используешь будущие данные
- Survivorship bias — тестируешь на монетах, которые не умерли
- Overfitting — стратегия запомнила прошлое, не нашла паттерн

**Как проверить:** Walk-forward analysis (skill `backtesting-frameworks`).  
Train на 2020-2022, test ТОЛЬКО на 2023-2024. Никогда не смотреть на test до конца.

---

### Допущение #2 (ВЫСОКИЙ РИСК)
> "Funding Rate Arbitrage — это безрисковый доход"

**Риск:** Частично ложное.  
**Почему частично:**
- В медвежьем рынке funding становится отрицательным (шортисты платят лонгам)
- 2022 год: funding на BTC был отрицательным суммарно 40% времени → стратегия убыточна
- Exchange risk: держишь капитал на бирже (FTX коллапс = всё потерял)

**Как проверить:** Протестировать стратегию на данных 2022 года специально.

---

### Допущение #3 (СРЕДНИЙ РИСК)
> "1-2 человека могут поддерживать несколько ботов одновременно"

**Риск:** Реальная угроза операционного выгорания.  
**Что происходит на практике:**
- Бот #1 упал в 3 часа ночи → нет алерта → пропустили движение
- Бот #2 открыл позицию в логике ошибки → висит убыточная сделка
- Рынок резко упал → нужно ручное вмешательство сразу

**Как проверить:** Запустить сначала ОДИН бот на малом капитале. Понять операционную нагрузку.

---

### Допущение #4 (КРИТИЧЕСКОЕ — часто игнорируют)
> "Плечо 3x — это безопасно"

**Риск:** Математически опасно при высокой волатильности крипто.

```
BTC. Волатильность 60-80% годовых = ~4-5% за день (обычно).
При плече 3x: движение цены 33% = ликвидация.
BTC за 1 день упал на 30%+? → Mai 2021: -30% за 24ч. → Да, случается.

Вывод: Даже 2x плечо требует стопа ДО ликвидационной цены.
```

**Как проверить:** Посчитать расстояние от точки входа до ликвидационной цены.  
Правило: стоп всегда на 50%+ выше цены ликвидации.

---

### Допущение #5
> "Биржа всегда будет работать"

**FTX (ноябрь 2022):** $8 млрд клиентских средств — исчезло.  
**Binance (2023):** SEC иск, временные проблемы с выводом USD.

**Митигация:** Никогда не держать на бирже более 30% капитала одновременно.

## ШАГ 5 — REVERSE BRAINSTORMING (Как гарантированно всё слить)

*Инверсия: сначала думаем как потерять деньги, потом разворачиваем.*

| Как потерять всё | Разворот → Как не потерять |
|-----------------|---------------------------|
| Поставить 20x плечо с первого дня | Максимум 2-3x, наращивать постепенно |
| Доверять только бэктесту без бумажной торговли | Минимум 2 недели paper trading перед реальными деньгами |
| Держать весь капитал на одной бирже | Разделить: Binance + Bybit + 30% в холодном кошельке |
| Не ставить стопы ("сейчас отрастёт") | Хардкодированные стопы в коде, не опциональные |
| Оптимизировать параметры на полных данных | Train/Test split. Test set — священен, не трогать |
| Не следить за ботом первые 2 недели | Ежедневный 5-минутный чек. Telegram алерты |
| Запустить все 3 бота одновременно | Один бот → стабилизировать → второй → стабилизировать → третий |
| API ключ с правом вывода | Только Trade + Order права. Никогда Withdraw |
| Игнорировать on-chain метрики | Funding rate + OI + exchange flow как фильтры |
| "Я разберусь с рисками потом" | Risk Register (ниже) до первой сделки |

## ШАГ 6 — OODA LOOP (Как адаптироваться быстрее рынка)

Рынок меняется. Стратегия, которая работала 6 месяцев, может перестать.  
OODA — это темп адаптации:

```
OBSERVE (Наблюдать — еженедельно)
  • P&L каждого бота по отдельности
  • Sharpe за последние 30 дней (если < 0.5 → что-то не так)
  • Funding rates: тренд вверх или вниз?
  • Режим рынка: trader-regime → bull/bear/range/volatile?
  • Сколько раз сработал circuit breaker за неделю?

ORIENT (Осмыслить — интерпретировать данные)
  • Почему этот бот убыточен? Изменился режим рынка?
  • Funding rate упал → Funding Bot нужно ставить на паузу?
  • Trend Bot даёт больше сделок, но меньший RR → overfitting?
  • Корреляция ботов между собой выросла → нет реальной диверсификации

DECIDE (Решить — небольшие, обратимые ставки)
  • Уменьшить размер позиции Trend Bot на 50% пока не прояснится
  • Приостановить Funding Bot, пока funding не вернётся к 0.01%+
  • Добавить режимный фильтр: Trend Bot активен ТОЛЬКО при ADX > 25

ACT (Действовать — и сразу наблюдать результат)
  • Внедрить изменение
  • Следующий цикл OODA через 2 недели
  • Не менять несколько параметров одновременно (не поймёшь, что помогло)
```

**Ключевой принцип OODA:** Не ждать "идеального момента" для изменений.  
Действовать малыми ставками. Кто адаптируется быстрее рынка — тот выигрывает.

## ШАГ 7 — СТРАТЕГИИ: ОППОРТЬЮНИТИ VS РЕАЛЬНОСТЬ

Применяем "сильный аргумент ПРОТИВ" каждой стратегии (thinking partner роль):

### Trend Following — Сильный аргумент против:
> "Крипто тренды существуют, но они коротки и непредсказуемы по началу.  
> Ты войдёшь в тренд на 30-50% его длины. Остальное — whipsaws."

**Ответ-митигация:** ADX > 25 убирает большинство ложных сигналов.  
Принимаем как часть стратегии: не все входы будут в начале тренда.

### Funding Rate Arbitrage — Сильный аргумент против:
> "Это НЕ безрисковый арбитраж. Это ставка на продолжение бычьего рынка.  
> В 2022 году за 12 месяцев стратегия была бы убыточна."

**Ответ-митигация:** Жёсткий стоп при funding < 0.003% per 8h. Не держать позицию "сквозь" медвежий рынок.

### Grid Bot — Сильный аргумент против:
> "При тренде вниз сетка набирает убыточные лонги.  
> Классический grid bot потерял 60-80% в 2022 медвежьем рынке."

**Ответ-митигация:** Активация ТОЛЬКО при ADX < 20 (боковик). Принудительное закрытие при выходе за диапазон.

## ШАГ 9 — СХОДИМОСТЬ (Что делаем ПЕРВЫМ)

По итогам Brainstorming + Risk Assessment:

```
ПРИОРИТЕТ 1 (Неделя 1):
  → Валидировать Assumption #1 (backtest ≠ reality)
    Конкретно: запустить backtesting-frameworks, провести walk-forward на 2022-2024
    Если Sharpe < 1.0 на out-of-sample → не идти дальше с этой стратегией

ПРИОРИТЕТ 2 (Неделя 2):
  → Внедрить Risk Register R1-R4 ДО первого реального ордера
    Конкретно: circuit breaker, spend limits из llm-trading-agent-security
    API ключ без Withdraw прав, IP whitelist

ПРИОРИТЕТ 3 (Неделя 3):
  → Запустить ОДИН бот (Funding Rate) на $500 тестового капитала
    Не Trend Bot (сложнее), не Grid (нужен боковик)
    Funding Bot проще, понятнее, легче дебажить

ПРИОРИТЕТ 4 (Неделя 5+):
  → OODA первый цикл: что наблюдаем через 2 недели live?
    → Только после стабильной работы Funding Bot → добавить Trend Bot
```

## ШАГ 10 — РОЛИ ДЛЯ 1 ЧЕЛОВЕКА (Пересмотр через OODA)

Соло — это не "делаю всё хуже". Это "убираю трение между ролями":

```
ПОНЕДЕЛЬНИК — Quant (30 мин)
  Смотрю метрики ботов за неделю.
  Запускаю trader-risk: проверяю VaR и Sharpe.
  Нужно менять стратегию? Записываю гипотезу.

СРЕДА — Developer (по необходимости)
  Если есть баг или улучшение — фиксирую.
  Никогда не деплою изменения в пятницу перед выходными.

ПЯТНИЦА — Risk Manager (15 мин)
  Открываю Risk Register. Какой статус R1-R12?
  Проверяю баланс биржи. Withdrawal прибыли если > 20% от начального депозита.

ВОСКРЕСЕНЬЕ — Research (1 час, по желанию)
  Читаю CryptoQuant / Deribit Insights / ArXiv.
  Есть новая идея? Добавляю в Opportunity Solution Tree.
  Не внедряю ничего не протестировав на paper trading.
```
